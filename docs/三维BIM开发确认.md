# 三维 BIM 开发确认

确认时间：2026-04-24

确认结论：

`确认进入开发。`

本轮开发以“三维 BIM 可视化交付底座验收收口”为目标，优先保障现有三维全屏能力稳定、可验收、可追溯。4D、5D、AI 助理、扫描建模闭环不纳入本轮主线开发，只保留后续扩展规划；BIM+GIS 不纳入本轮，也不做前端扩展位预留。

## 一、本轮开发目标

本轮不做“大而全 BIM 平台”，只确认开发以下目标：

- 稳定模型资产加载链路。
- 完善模型资产选择体验。
- 完善构件属性、KKS、设备台账、运行参数、文档、测点的数据闭环。
- 强化导航图、结构树、设备定位、场景方案等已有能力的验收可用性。
- 补齐数据来源、模拟数据标识、资产覆盖矩阵和验收矩阵。

最终交付口径：

`当前版本交付一个可验证、可演示、可扩展的三维数字交付底座，而不是完整 BIM5D 或全厂数字孪生平台。`

## 二、确认开发范围

| --- | --- | --- | --- | --- |

| BIM-P0-003 | 构件属性数据闭环 | `ObjectInfoPanel.vue` 已有 KKS、台账、结构参数、运行参数、文档、测点 | 增加数据来源、更新时间、模拟数据标识、缺失项提示；梳理字段白名单 | 选中构件后能说明每类数据来源；缺失数据不会误判为功能缺失 |

## 三、不纳入本轮开发范围

以下能力不进入本轮验收主线：

| 能力 | 不纳入原因 | 后续进入条件 |
| --- | --- | --- |
| 4D 进度模拟 | 缺少 WBS、计划任务、构件绑定数据 | 有项目计划数据和构件绑定关系 |
| 5D 成本/物资统计 | 缺少成本、物资、合同、工程量数据 | 有成本科目和物资台账 |
| BIM+GIS | 缺少 GIS 底图、坐标系、厂区空间数据 | 不纳入本轮，不预留前端扩展位 |
| AI 助理 | 数据索引和权限边界未确定 | 文档、模型、KKS、台账统一索引完成 |
| 扫描建模更新闭环 | 属于模型生产与发布流程，不是前端单项能力 | 模型侧提供标准发布包、版本台账和更新流程 |
| 完整模型版本几何对比 | 需要稳定构件 ID 和差异算法 | 模型构件 ID 稳定并有版本数据 |

## 四、当前文档修正要求

旧文档中部分判断已过时，本轮开发前需要同步修正口径。

| 文档/口径 | 当前问题 | 修正方向 |
| --- | --- | --- |
| “缺少图形化导航图” | 当前代码已有 `NavigationMapPanel.vue` | 改为“导航图增强与验收配置补齐” |
| “fullscreen 未展示设备台账/参数” | 当前 `ObjectInfoPanel.vue` 已有设备台账、结构参数、运行参数 | 改为“字段标准、真实数据来源、更新时间、模拟数据标识未闭环” |
| “全厂级场景已完成” | 缺少资产覆盖清单 | 改为“具备全厂扩展能力，当前交付范围以资产覆盖矩阵为准” |
| “定期扫描建模更新” | 缺少模型生产流程 | 拆成模型侧生产流程和前端发布包消费能力 |

## 五、开发顺序

### 第一批：稳定性和验收收口

1. 模型资源鉴权加载稳定。
2. 模型资产选择体验统一。
3. 构件属性数据来源和缺失提示。
4. 场景保存恢复校验。
5. 验收矩阵和资产覆盖矩阵模板。

### 第二批：交互增强

1. 导航图增强。
2. 结构树绑定状态增强。
3. 文档联动预览。
4. 告警定位与状态着色。

### 第三批：扩展预留

1. 模型版本元数据展示。
2. 发布包版本信息展示。
3. 4D/5D 数据扩展位预留。

## 六、验收入口

本轮开发完成后，以以下页面和材料验收：

- `/visualization/3d-fullscreen`
- 模型管理页面
- 数据管理/KKS 台账
- 文档管理/文档预览
- 资产覆盖矩阵
- 三维 BIM 验收矩阵
- 浏览器 Network 请求截图
- 三维页面功能截图或录屏

## 七、风险确认

| 风险 | 影响 | 处理方式 |
| --- | --- | --- |
| 后台接口字段不完整 | 台账、运行参数、版本信息无法完整展示 | 前端先展示已有字段，并明确缺失项和数据来源 |
| 模型资产没有稳定构件 ID | 构件绑定、版本对比、问题定位不稳定 | 本轮只做绑定状态和现有 UUID 能力，不承诺跨版本几何对比 |
| 全厂资产清单缺失 | 无法按全厂级验收 | 资产覆盖矩阵作为验收前置物 |
| 模型资源鉴权不统一 | GLB、贴图、LOD、缩略图可能加载失败 | P0 优先统一资产加载链路 |
| 需求继续扩张 | P0 交付延期 | 4D/5D/AI 冻结到 P2 或专项；BIM+GIS 不做本轮预留 |

## 八、确认结果

本轮确认进入开发的范围为：

`P0 验收收口 + P1 体验增强。`

本轮不确认进入开发的范围为：

`完整 4D/5D、BIM+GIS、AI 助理、扫描建模生产闭环、跨版本几何对比。`

其中 BIM+GIS 只保留为专项方向说明，不在当前前端项目包 schema 和页面中预留扩展入口。

后续所有新增需求必须满足以下条件后再进入开发：

- 有明确业务价值。
- 有数据来源。
- 有接口或模型产物。
- 有验收标准。
- 有责任人。
- 不影响 P0 验收收口。
