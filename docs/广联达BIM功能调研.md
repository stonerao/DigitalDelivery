# 广联达 BIM 功能调研

检索时间：2026-04-24

## 调研范围

本调研面向市面公开可访问的广联达 BIM 相关产品与平台能力，重点关注可被数字交付、三维可视化、模型管理、项目协同、数据管理模块借鉴的功能。信息来源优先采用广联达官网、BIMFACE、BIMMAKE、协筑、数字孪生平台等官方公开资料；BIM5D 独立产品官网资料较少，施工 5D 功能参考 App Store 官方应用页与第三方公开教程摘要，需在正式竞品对标前再做客户侧或产品演示验证。

## 关键功能拆解

### 1. 模型生产与模型资产

广联达的 BIMMAKE 覆盖施工建模、场地策划和深化设计三个典型入口。公开资料显示，其施工建模强调通过建模、导模、翻模快速形成施工 BIM 模型；场地策划强调构件库、策划算法、显示引擎、施工图和工程量统计；深化设计强调跨阶段模型互通，并输出场布、钢筋、模架、砌体等深化图纸和材料量。

对本项目可借鉴的功能：

- 模型资产库应支持模型缩略图、模型类型、文件大小、上传时间、版本、来源系统、关联项目等基础元数据。
- 模型管理应区分原始文件、轻量化文件、缩略图、转换状态、转换日志。
- 模型上传后可增加“生成缩略图”“生成轻量化模型”“提取构件树/属性”三个异步任务。
- 资产选择弹窗不应只是下拉框，建议采用卡片列表，展示缩略图、名称、专业、时间、大小和状态。

### 2. 轻量化浏览与 Web BIM 能力

BIMFACE 定位为 BIM 轻量化引擎和二次开发平台。公开资料显示其支持 50+ 文件格式云端转换，支持手机、平板等多终端网页浏览，能力包括模型预览、测量、剖切、路径漫游、BIM+GIS、模型效果渲染以及开放 API。

对本项目可借鉴的功能：

- 三维查看器基础工具栏建议覆盖：模型树、构件属性、测量、剖切、爆炸/隔离/隐藏、视点保存、漫游、复位、截图。
- 支持大模型时应引入轻量化模型、流式加载或分包加载策略，避免直接下载完整 GLB 后再渲染。
- 模型接口要区分下载鉴权和预览鉴权，下载接口需要携带 token/cookie，避免全屏页模型加载出现匿名请求失败。
- 若后续需要二次开发平台能力，可抽象 Viewer SDK 层，避免业务页面直接耦合 three.js 加载器和后端接口。

### 3. 设计协同与数字化交付

广联达 BIM 设计协同平台面向设计企业，公开资料显示其集成二三维一体化协同设计、设计项目管理、设计校审、数字化交付和企业资源管理，并支持广联达数维设计软件、Revit、CAD 等工具。广联达建筑设计方案资料还强调统一数据中心、项目策划、任务计划、协同设计、模型轻量化、图模校审、交付归档，以及跨专业、跨地域协同。

对本项目可借鉴的功能：

- 交付资料应围绕项目建立统一数据中心，支持模型、图纸、文档、KKS、设备、点位、摄像头等对象统一归档。
- 模型与图纸需要建立关联关系，支持从构件跳转图纸、从图纸定位模型构件。
- 设计/交付版本应可追溯，至少记录版本号、变更说明、上传人、上传时间、审核状态。
- 可增加“交付检查清单”，用于检查模型、文档、KKS、设备台账、图纸目录是否齐套。

### 4. 文档、流程与 BIM 协作

协筑的公开资料显示，其以文档管理、任务流程、BIM 协作、团队沟通为核心，支持图档集中管理、权限控制、自动版本管理、在线预览、批注评论、版本对比、流程任务、消息提醒、移动办公等。BIM 协作部分包括全专业模型集成浏览、自由剖切、过滤筛选、构件属性查看、漫游、基于视点的问题批注和任务跟踪。

对本项目可借鉴的功能：

- 文档管理模块可增加在线批注、版本对比、文件订阅、操作留痕。
- 模型查看器可增加“问题批注”：保存相机视角、构件 ID、截图、问题描述、责任人、状态。
- 任务流程应能从模型构件、文档、图纸、KKS 数据行直接发起，形成问题闭环。
- 项目协同需要统一消息模型，覆盖文件上传、模型更新、批注、任务状态变更。

### 5. 施工 4D/5D 管理

BIM5D 公开资料显示其以 BIM 平台为核心，集成全专业模型，并以模型为载体关联进度、合同、成本、质量、安全、图纸、物料等信息，用于决策和精细管理。公开产品介绍中常见功能包括施工模拟、进度控制、成本控制、质量安全管理、动画演示、物资提量与三算对比。

对本项目可借鉴的功能：

- 当前数字交付平台可先实现“模型 + 资料 + 资产”的 3D 交付底座，再逐步扩展 4D 进度和 5D 成本。
- 进度维度可通过任务/WBS 绑定模型构件或区域，实现按时间轴查看施工状态。
- 成本/物资维度可从 KKS、设备台账、材料台账、模型构件属性中提取统计口径。
- 质量安全可基于模型空间位置发起问题，并与照片、视频、整改记录关联。

### 6. BIM+智慧工地与 IoT

广联达 BIM+智慧工地数据决策系统公开资料显示，其利用 IoT、BIM、大数据、AI 实时采集现场数据并分析建模，覆盖企业项目决策分析、生产、安全、机械、质量、技术、物资、分包、商务等业务；同时包含 AI 助理、AI 知识平台、大型机械管理、100 余类 IoT 设备接入和 APP/WEB/微信多端协同。

对本项目可借鉴的功能：

- 摄像头、传感器、点位等数据应与三维空间坐标绑定，支持在 3D 场景中直接查看实时数据。
- 智慧工地能力应采用可插拔数据源，例如视频平台、门禁、环境监测、塔吊、升降机、人员定位。
- 数据看板建议区分企业级、项目级、区域级、设备级四层视角。
- AI 能力可优先落地在资料问答、规范检索、风险提示、日报生成、问题归类等低耦合场景。

### 7. 数字孪生与 BIM+GIS

广联达数字孪生平台公开资料显示，其提供跨平台、无插件二三维一体化渲染，支持 BIM+GIS 多源空间数据汇聚、轻量化、发布共享、场景搭建、空间分析、特效可视化、无人机建模、吊装规划和高真实感渲染。

对本项目可借鉴的功能：

- 三维全屏场景可从单模型查看升级为“场景管理”：底图、模型、点位、摄像头、标注、图层、路径、视点统一管理。
- 对厂区/园区/城市级场景，需要同时支持 BIM 模型和 GIS 图层，模型定位需具备坐标系转换能力。
- 场景编辑器应支持图层开关、资产拖拽、标注、视点保存、漫游路线、动画播放。
- 对大型场景需考虑瓦片化、LOD、实例化渲染和按需加载。

## 功能落地拆分建议

### 阶段一：模型资产可用

- 模型列表、搜索、缩略图、基础元数据。
- 模型上传、下载、预览鉴权统一。
- 模型状态：上传中、转换中、可预览、失败。
- 三维全屏资产选择支持卡片列表和多选。

### 阶段二：BIM 查看器增强

- 模型树、构件属性、测量、剖切、隐藏/隔离。
- 视点保存、截图、漫游路径。
- 文档、KKS、设备和模型构件关联。

### 阶段三：协同与问题闭环

- 模型批注、图纸批注、问题任务。
- 任务责任人、状态、截止时间、整改附件。
- 操作留痕、消息提醒、版本对比。

### 阶段四：数字孪生扩展

- 场景图层、GIS 底图、点位/摄像头/IoT 数据接入。
- 进度时间轴、施工模拟。
- 数据驾驶舱、AI 辅助分析。
