# BIM三维点位与摄像头数据模型设计

日期：2026-03-25

## 1. 文档目标

本文档用于定义 BIM 三维平台中的点位、摄像头点位、关联样式、运行态状态和持久化结构，作为后续点位系统、摄像头系统、实时数据绑定和配置工具实现的统一依据。

本文档对应 [技术分册.md](e:/项目/数字交付/DigitalDelivery/docs/技术分册.md#L1) 中“点位系统”和“摄像头系统”的进一步展开。

## 2. 设计原则

数据模型设计遵循以下原则：

- 统一建模：所有点位都基于同一基础模型扩展
- 运行态与配置态分离：配置定义与实时状态不混写
- 构件绑定优先：优先绑定 `objectUuid`
- 工程持久化优先：不依赖页面内存态和本地存储作为唯一来源
- 扩展兼容：后续可扩展视频、告警、脚本触发、权限控制

## 3. 模型总览

建议拆成四类核心模型：

1. `SceneAnchorDefinition`
2. `CameraAnchorDefinition`
3. `AnchorRuntimeState`
4. `AnchorStyleDefinition`

说明：

- `Definition` 用于工程持久化
- `RuntimeState` 用于运行时实时变化
- `StyleDefinition` 用于统一视觉配置

## 4. 基础点位模型

## 4.1 SceneAnchorDefinition

建议字段如下。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 点位唯一标识 |
| `type` | string | 点位类型 |
| `name` | string | 点位名称 |
| `code` | string | 点位编码，可选 |
| `description` | string | 说明 |
| `anchorMode` | `object` \| `world` | 绑定方式 |
| `objectUuid` | string | 绑定构件 UUID |
| `worldPosition` | `[number, number, number]` | 世界坐标 |
| `localPosition` | `[number, number, number]` | 相对构件局部坐标，可选 |
| `offset` | `[number, number, number]` | 展示偏移 |
| `rotation` | `[number, number, number]` | 旋转 |
| `scale` | number | 缩放 |
| `visible` | boolean | 默认是否显示 |
| `locked` | boolean | 是否锁定编辑 |
| `groupId` | string | 所属分组 |
| `tags` | string[] | 标签 |
| `styleId` | string | 样式 ID |
| `businessBinding` | object | 业务绑定 |
| `payload` | object | 类型专属配置 |
| `createdAt` | string | 创建时间 |
| `updatedAt` | string | 更新时间 |
| `createdBy` | string | 创建人 |
| `updatedBy` | string | 修改人 |

## 4.2 type 类型建议

建议支持以下类型：

- `device-point`
- `camera-point`
- `label-point`
- `measurement-point`
- `warning-point`
- `custom-point`

说明：

- `camera-point` 在模型上属于点位的一种，但有专门扩展字段
- `measurement-point` 用于测点、仪表、实时值挂点，不等于几何测量结果

## 4.3 businessBinding 结构建议

```json
{
  "nodeId": "sys-1-1",
  "kks": "10LAC10AP001",
  "deviceId": "dev-001",
  "tag": "PT-101",
  "bindingType": "device"
}
```

建议字段：

- `nodeId`
- `kks`
- `deviceId`
- `tag`
- `bindingType`

## 5. 摄像头点位模型

## 5.1 CameraAnchorDefinition

摄像头点位建议继承基础点位模型，并扩展摄像头专有字段。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `cameraCode` | string | 摄像头编码 |
| `cameraName` | string | 摄像头名称 |
| `streamType` | string | 视频流类型，第一阶段为 `flv` |
| `streamUrl` | string | 视频流地址 |
| `poster` | string | 封面图 |
| `vendor` | string | 厂商 |
| `ptzEnabled` | boolean | 是否支持云台 |
| `audioEnabled` | boolean | 是否有音频 |
| `defaultWindowMode` | `dialog` \| `drawer` \| `panel` | 默认打开方式 |
| `bindDeviceKks` | string | 关联设备 KKS |
| `bindObjectUuid` | string | 关联模型对象 UUID |
| `statusMapping` | object | 状态映射 |
| `authConfig` | object | 鉴权配置 |
| `payload` | object | 扩展参数 |

## 5.2 示例

```json
{
  "id": "camera-anchor-001",
  "type": "camera-point",
  "name": "给水泵A摄像头",
  "anchorMode": "object",

  "visible": true,
  "groupId": "camera-group-1",
  "styleId": "camera-online",
  "businessBinding": {
    "nodeId": "sys-1-1",
    "kks": "10LAC10AP001",
    "bindingType": "device"
  },
  "cameraCode": "CAM-FDP-A-001",
  "cameraName": "给水泵A现场摄像头",
  "streamType": "flv",
  "streamUrl": "https://example.com/live/fdp-a.flv",
  "poster": "/assets/camera/fdp-a.jpg",
  "vendor": "DemoVendor",
  "ptzEnabled": false,
  "audioEnabled": false,
  "defaultWindowMode": "dialog",
  "bindDeviceKks": "10LAC10AP001",
  "bindObjectUuid": "7cb2f3ab-xxxx"
}
```

## 6. 样式模型

## 6.1 AnchorStyleDefinition

建议将点位样式独立配置，避免每个点位重复保存大量视觉字段。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 样式 ID |
| `name` | string | 样式名称 |
| `iconType` | string | 图标类型 |
| `iconUrl` | string | 自定义图标地址 |
| `color` | string | 主色 |
| `backgroundColor` | string | 背景色 |
| `borderColor` | string | 边框色 |
| `size` | number | 图标尺寸 |
| `labelVisible` | boolean | 标签是否显示 |
| `labelColor` | string | 标签颜色 |
| `labelBgColor` | string | 标签背景色 |
| `billboard` | boolean | 是否始终朝向相机 |
| `pulse` | boolean | 是否闪烁 |
| `zIndex` | number | 层级 |

## 6.2 样式分层建议

建议样式来源按以下优先级覆盖：

1. 点位实例覆盖
2. 类型默认样式
3. 工程全局样式

## 7. 运行时状态模型

## 7.1 AnchorRuntimeState

配置态与运行态应分离。

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 点位 ID |
| `resolvedWorldPosition` | `[number, number, number]` | 解析后的场景坐标 |
| `visible` | boolean | 当前是否显示 |
| `selected` | boolean | 当前是否选中 |
| `hovered` | boolean | 当前是否悬停 |
| `status` | string | 当前状态 |
| `value` | string \| number | 当前值 |
| `displayText` | string | 展示文本 |
| `lastUpdatedAt` | string | 最后更新时间 |
| `error` | string | 异常信息 |

说明：

- `resolvedWorldPosition` 由引擎根据对象绑定或世界坐标计算
- 运行态值不应直接写回配置文件

## 7.2 CameraRuntimeState

建议字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 摄像头点位 ID |
| `online` | boolean | 是否在线 |
| `streamConnected` | boolean | 流是否接通 |
| `playing` | boolean | 是否正在播放 |
| `windowVisible` | boolean | 视频窗口是否显示 |
| `currentStreamUrl` | string | 当前播放地址 |
| `lastHeartbeatAt` | string | 最后心跳时间 |
| `error` | string | 播放或连接错误 |

## 8. 点位分组模型

建议补充 `AnchorGroupDefinition`，用于批量控制显隐和分类管理。

建议字段：

- `id`
- `name`
- `type`
- `visible`
- `parentId`
- `order`
- `styleId`
- `description`

用途：

- 分组控制点位显隐
- 配置端目录展示
- 权限控制与业务分类

## 9. 持久化文件建议

建议在工程包中至少包含以下文件。

### anchors.json

存放非摄像头点位定义：

- `device-point`
- `label-point`
- `measurement-point`
- `warning-point`
- `custom-point`

### cameras.json

存放摄像头点位定义。

### anchor-styles.json

存放样式定义。

### anchor-groups.json

存放点位分组定义。

## 11. 编辑能力映射

为了支持后续配置工具和运行时编辑，建议将能力映射到模型字段。

### 创建点位

需要填写：

- 类型
- 名称
- 绑定模式
- 绑定目标
- 初始样式
- 所属分组

### 编辑点位

需要支持：

- 修改名称、样式、分组
- 修改绑定对象或坐标
- 修改偏移和显隐
- 修改业务绑定

### 摄像头编辑

需要支持：

- 填写编码和名称
- 配置流类型与地址
- 配置封面图
- 配置默认弹窗方式
- 配置是否启用音频和云台

## 12. 与运行时交互关系

建议运行时按以下方式消费数据模型。

### 点位加载

1. 读取 `anchors.json`
2. 根据 `anchorMode` 解析场景位置
3. 挂载场景锚点渲染对象
4. 绑定点击事件与悬停事件
5. 根据实时数据刷新运行态状态

### 摄像头加载

1. 读取 `cameras.json`
2. 渲染摄像头点位
3. 点击后根据 `streamType` 选择播放器
4. 打开视频窗口并同步播放状态

## 13. 接口与校验建议

建议后续为配置工具和运行时建立统一校验逻辑。

### 基础校验

- `id` 不能为空且全局唯一
- `type` 必须在支持列表内
- `anchorMode=object` 时必须有 `objectUuid`
- `anchorMode=world` 时必须有 `worldPosition`
- `styleId` 引用必须存在

### 摄像头校验

- `streamType` 必须合法
- `streamUrl` 不能为空
- 同一工程内 `cameraCode` 建议唯一

### 业务绑定校验

- `nodeId`、`kks`、`deviceId` 至少有一个可识别字段
- 绑定设备时应尽量校验 KKS 是否存在

## 14. 与现有代码的衔接建议

当前 [fullscreen.vue](e:/项目/数字交付/DigitalDelivery/src/views/visualization/viewer3d/fullscreen.vue#L1) 已有以下可复用能力：

- 构件 UUID 选中与定位链路
- 业务 KKS 与设备 profile 绑定链路
- 测点联动展示链路
- 场景方案本地持久化链路

建议衔接方式：

- 点位绑定优先复用现有 `uuid` 选择链路
- 摄像头点位纳入统一点位系统，不单独造数据结构体系
- 测点数据展示与 `measurement-point` 逐步统一
- 场景方案后续可扩展为保存点位显隐和摄像头窗口状态
