# 三维全屏场景问题清单与整改建议（简版）

更新时间：2026-04-24

适用对象：项目汇报、需求对齐、阶段验收沟通。

## 一、结论

当前 `/visualization/3d-fullscreen` 已具备较完整的三维巡视平台能力，但按“全厂级三维场景”需求验收，**暂不能判定为完全满足**。

当前判断：

- 第 1 条 全厂场景漫游与设备/台账展示：**部分满足**
- 第 2 条 导航图与快速定位：**部分满足**
- 第 3 条 三维巡视与实时渲染：**基本满足**
- 第 4 条 扫描建模定期更新：**不满足**

当前版本更准确的定位应为：

`已完成三维巡视平台能力建设和局部场景展示，距离全厂级完整交付仍有差距。`

## 二、当前已具备能力

1. 三维模型加载、实时渲染、画质切换、弱机降级
2. 第一人称漫游、视角书签、预设视角、对象拾取与定位
3. 结构树导航、系统定位、KKS 定位
4. 点位、摄像头、测量、剖切、截图
5. 对象与 KKS、文档、测点的绑定能力
6. 部分实时数据、视频联动与运行时事件接入能力

## 三、核心问题清单

### 1. 缺图形化导航图

现状：

- 现在只有树形导航、系统定位、KKS 定位、书签定位。
- 没有厂区导航图、小地图、区域总览图。

影响：

- 不能满足“配置可以瞬间定位的导航图”的要求。

### 2. fullscreen 未完整展示设备参数和台账

现状：

- fullscreen 当前主要展示 KKS、基本属性、文件和测点。
- 未完整展示台账号、厂家、型号、额定参数、结构参数、运行参数。

影响：

- 不满足“显示主机和重要辅机设备参数，包括台账信息”的要求。

### 3. 无法证明当前场景已覆盖全厂资产

现状：

- 需求要求覆盖办公楼、厂房、道路、绿植、路灯、宣传栏、雕像、主辅机、管道、阀门、仪表等。
- 当前仓库没有完整的全厂场景资产覆盖清单可核对。

影响：

- 不能证明当前交付的是“全厂级场景”，只能证明平台有展示能力。

### 4. 当前更像局部区域场景

现状：

- 文档和示例数据更多体现为“汽机房模型”“锅炉系统”“控制室”“冷却塔”等局部区域或局部系统。

影响：

- 现阶段更像区域级示范场景，不是完整全厂总场景。

### 5. 扫描建模更新链路未落地

现状：

- 当前只有运行时消费 `scene-manifest`、质量档、LOD 的能力。
- 离线扫描建模、贴图整理、LOD 生成、压缩减面、发布产物流程未完成。

影响：

- 不满足“根据建设进度定期扫描建模更新”的要求。

### 6. 状态展示仍是部分满足

现状：

- 已有部分业务参数、测点和实时数据接入基础。
- 但真实实时数据覆盖、协议完整性、状态同步能力仍不足。

影响：

- 不能直接认定为“能够全面掌握厂内状态”。

## 四、整改优先级

### P0：必须先补

1. 明确当前交付范围：局部区域还是全厂级
2. 补 fullscreen 的设备台账和参数展示
3. 输出全厂资产覆盖清单

### P1：本阶段建议补齐

1. 增加图形化导航图/小地图
2. 打通主机和重要辅机的真实参数展示
3. 建立导航图与区域/系统/视角联动

### P2：后续能力建设

1. 建设离线资产处理流水线
2. 打通扫描建模更新流程
3. 完善版本管理、发布产物和资源目录规范

## 五、建议对外口径

建议当前统一口径为：

`当前系统已完成三维巡视平台与局部场景能力建设，已具备三维浏览、漫游、定位、点位、测量、摄像头、业务绑定等核心基础能力；若按全厂级需求验收，仍需补齐导航图、完整参数台账展示、全厂资产覆盖和扫描建模更新链路。`
