## 文档管理页面对接接口及说明

目标页面：`/handover/documents`

### 1）页面初始化与筛选

- 文档列表：`GET /api/handover/documents/list`
    - 对应筛选参数：`keyword/searchMode/types/updatedFrom/updatedTo/kksKeyword/nodeId/folderId/recycleBin/page/size`
    - 页面行为：点击“应用”、切换目录、切换回收站时重新拉取列表。
- 目录树：`GET /api/handover/docFolders/tree`
    - 页面行为：进入页面时加载目录树，目录拖拽/导入后刷新目录树。

### 2）文档操作

- 上传文档：`POST /api/handover/documents/upload`（`multipart/form-data`）
    - 参数：`files[]/folderId/transferDepth`（关联字段可按需追加）
- 重命名/关联编辑：`POST /api/handover/documents/update`
    - 用于“重命名”“关联关系保存”。
- 移动文档：`POST /api/handover/documents/move`
    - 参数：`ids[]/targetFolderId`
- 删除到回收站：`POST /api/handover/documents/delete`
- 回收站恢复：`POST /api/handover/documents/restore`
- 彻底删除：`POST /api/handover/documents/purge`
- 单文档下载地址：`GET /api/handover/documents/{id}/downloadUrl`
- 批量下载：`POST /api/handover/documents/batchDownload`

### 3）版本管理

- 查询版本：`GET /api/handover/documents/{id}/versions`
- 上传新版本：`POST /api/handover/documents/{id}/versions/upload`（`multipart/form-data`）
- 回滚版本：`POST /api/handover/documents/{id}/versions/rollback`

### 4）目录管理

- 新建目录：`POST /api/handover/docFolders/create`
- 重命名目录：`POST /api/handover/docFolders/rename`
- 移动目录：`POST /api/handover/docFolders/move`
- 删除目录：`POST /api/handover/docFolders/delete`
- 导入目录 JSON：`POST /api/handover/docFolders/import`
- 导出目录 JSON：`GET /api/handover/docFolders/export`

### 5）对象字典

- 系统节点选项：`GET /api/handover/systemNodes/options`
- KKS 选项：`GET /api/handover/kks/options`
- 页面行为：用于筛选区“关联系统”下拉，以及“对象编码关联”弹窗中的系统/KKS选择器。

### 6）实现约定

- 页面统一按返回结构 `code/message/data` 解析，核心读取 `data.records`（列表）与 `data.url`（下载）。
- 列表与目录的写操作成功后，前端执行一次“重新拉取”以保证界面一致性。
- 对象详情展示仍保留前端回退逻辑；筛选与关联下拉已优先使用后端 options 接口。
