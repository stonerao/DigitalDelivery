
# 数据移交-模型管理 API

## 模型管理页面对接接口及说明

目标页面：`/handover/models`

### 1）页面初始化

- 模型列表：`GET /api/handover/models/list`
  页面行为：进入页面加载列表；上传/编辑/删除后刷新列表。

### 2）模型上传（异步任务）

- 上传模型：`POST /api/handover/models/upload`（`multipart/form-data`）
  前端提交：`files[]/lod`（`nodeIds[]/kksRefs[]` 视业务扩展）
- 上传任务状态：`GET /api/handover/models/upload/task`
  页面行为：上传后按 `taskId` 轮询任务状态，成功后刷新模型列表。

### 3）模型编辑与详情

- 模型详情：`GET /api/handover/models/detail?id={id}`
  页面行为：打开“查看”弹窗时请求详情，展示 `meta.author/meta.version` 等扩展字段。
- 更新模型：`POST /api/handover/models/update`
  页面行为：编辑模型名称与 LOD 后提交，成功后刷新列表。

### 4）模型删除

- 删除模型：`POST /api/handover/models/delete`
  页面行为：删除二次确认后提交 `ids[]`，成功后刷新列表。

### 5）实现约定

- 页面统一按 `code/message/data` 解包，列表读取 `data.records`。
- 任务轮询状态按关键字判断：`done/success/completed` 视为完成，`failed/error` 视为失败。
- 三维预览继续复用现有 `/visualization/3d-viewer` 路由，优先使用模型记录/详情返回的 `url`。

## 上传模型

**接口地址**:`/api/handover/models/upload`

**请求方式**:`POST`

**请求数据类型**:`application/x-www-form-urlencoded,application/json`

**响应数据类型**:`*/*`

**接口描述**:

**请求参数**:

**请求参数**:

| 参数名称 | 参数说明 | 请求类型    | 是否必须 | 数据类型 | schema |
| -------- | -------- | ----- | -------- | -------- | ------ |
|strings|string|body|true|array||
|modelUploadRequest|模型上传请求|body|true|ModelUploadRequest|ModelUploadRequest|
|&emsp;&emsp;lod|LOD级别||true|string||
|&emsp;&emsp;nodeIds|系统节点ID列表||false|array|string|
|&emsp;&emsp;kksRefs|KKS编码列表||false|array|string|

**响应状态**:

| 状态码 | 说明 | schema |
| -------- | -------- | ----- |
|200|OK|ResultUploadTaskResponse|

**响应参数**:

| 参数名称 | 参数说明 | 类型 | schema |
| -------- | -------- | ----- |----- |
|code||integer(int32)|integer(int32)|
|message||string||
|data||UploadTaskResponse|UploadTaskResponse|
|&emsp;&emsp;taskId||string||
|&emsp;&emsp;status||string||

**响应示例**:

```javascript
{
	"code": 0,
	"message": "",
	"data": {
		"taskId": "",
		"status": ""
	}
}
```
