# 数据移交-搜索导航接口设计

目标模块：`/search-nav`

对应前端实现：

- `src/views/searchNav/search/index.vue`
- `src/views/searchNav/navigation/index.vue`
- `src/views/searchNav/relationships/index.vue`
- `src/store/modules/ddLinkage.js`

## 1. 适用范围

当前搜索导航板块仅覆盖以下 3 个子模块：

- 搜索：`/search-nav/search`
- 系统导航：`/search-nav/navigation`
- 关联关系：`/search-nav/relationships`

说明：

- 目前搜索页实际仅支持搜索 `documents`、`models`、`devices`
- 页面上虽然有 `P&ID图纸` 选项，但现阶段前端逻辑将其并入 `documents` 一并查询，不是独立数据源

## 2. 总体数据域

建议后端围绕统一联动数据源提供以下 5 类核心数据：

- `systemTree`：系统/专业树
- `nodeMeta`：系统节点元数据
- `kksItems`：设备/KKS 台账
- `documents`：文档记录
- `models`：模型记录
- `relations`：显式关联关系

## 3. 核心数据模型

## 3.1 SystemTreeNode

系统导航左侧树直接使用该结构。

```json
{
  "id": "sys-1-1",
  "label": "给水泵系统（10LAC10）",
  "children": []
}
```

字段说明：

- `id`：系统节点主键
- `label`：节点显示名称
- `children`：子节点数组

## 3.2 NodeMeta

系统导航右侧详情依赖该结构。

```json
{
  "nodeId": "sys-1-1",
  "systemName": "给水泵系统",
  "discipline": "热机",
  "scope": "锅炉岛 / 给水泵与阀门回路",
  "owner": "专业负责人：张三",
  "description": "系统用于锅炉给水输送与压力维持。",
  "kksRoot": "10LAC10"
}
```

## 3.3 KksItem

```json
{
  "kks": "10LAC10AP001",
  "name": "主给水泵A",
  "type": "设备",
  "status": "待校验",
  "nodeId": "sys-1-1"
}
```

说明：

- 当前前端主要使用 `type === "设备"` 的记录
- `nodeId` 用于挂接系统节点

## 3.4 DocumentRecord

```json
{
  "id": "doc-2",
  "name": "设备数据表.xlsx",
  "size": 1258291,
  "updatedAt": "2024-01-14",
  "type": "EXCEL",
  "url": "",
  "folderId": "root",
  "nodeIds": ["sys-1-1"],
  "kksRefs": ["10LAC10AP001"]
}
```

说明：

- `nodeIds` 表示该文档关联的系统节点
- `kksRefs` 表示该文档关联的 KKS/设备

## 3.5 ModelRecord

```json
{
  "id": "m-1",
  "name": "锅炉系统模型",
  "url": "/model/test.glb",
  "lod": "LOD400",
  "components": 1254,
  "updatedAt": "2024-01-13",
  "nodeIds": ["sys-1-1"],
  "kksRefs": ["10LAC10"]
}
```

## 3.6 RelationRecord

```json
{
  "id": "r1",
  "type": "node_doc",
  "source": {
    "kind": "node",
    "id": "sys-1-1"
  },
  "target": {
    "kind": "doc",
    "id": "doc-2"
  },
  "updatedAt": "2026-01-01 10:12"
}
```

关系类型当前固定为：

```json
[
  { "label": "节点-文档", "value": "node_doc" },
  { "label": "节点-模型", "value": "node_model" },
  { "label": "KKS-文档", "value": "kks_doc" },
  { "label": "KKS-模型", "value": "kks_model" }
]
```

说明：

- `source.kind` 当前支持：`node`、`kks`
- `target.kind` 当前支持：`doc`、`model`
- 若后续要扩展为任意资源关联，可保留该结构继续扩展

## 4. 搜索模块接口

页面：`/search-nav/search`

当前搜索范围：

- `documents`
- `models`
- `devices`

页面中的 `pid` 目前并入 `documents`。

### 4.1 搜索接口

`GET /api/search-nav/search`

请求参数：

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `query` | string | 否 | 搜索关键字 |
| `scopeKeys` | string[] | 是 | 搜索范围，可选值：`documents`、`models`、`devices`、`pid` |

示例：

```json
{
  "query": "10LAC10AP001",
  "scopeKeys": ["documents", "models", "devices"]
}
```

返回结构：

```json
[
  {
    "id": "doc-2",
    "kind": "doc",
    "scope": "documents",
    "title": "设备数据表.xlsx",
    "type": "文档",
    "kks": "10LAC10AP001",
    "description": "关联系统：给水泵系统（10LAC10）"
  },
  {
    "id": "m-1",
    "kind": "model",
    "scope": "models",
    "title": "锅炉系统模型",
    "type": "三维模型",
    "kks": "10LAC10",
    "description": "关联系统：给水泵系统（10LAC10）"
  },
  {
    "id": "10LAC10AP001",
    "kind": "kks",
    "scope": "devices",
    "title": "主给水泵A（10LAC10AP001）",
    "type": "设备/台账",
    "kks": "10LAC10AP001",
    "description": "所属系统：给水泵系统（10LAC10）"
  }
]
```

字段说明：

- `kind`：前端跳转依据，当前可能为 `doc`、`model`、`kks`
- `scope`：搜索来源域，当前可能为 `documents`、`models`、`devices`
- `title`：列表标题
- `type`：展示用类型名称
- `kks`：可为空
- `description`：详情弹窗说明

## 5. 系统导航模块接口

页面：`/search-nav/navigation`

### 5.1 查询系统树

`GET /api/search-nav/system-tree`

返回结构：

```json
[
  {
    "id": "sys-1",
    "label": "锅炉系统",
    "children": [
      {
        "id": "sys-1-1",
        "label": "给水泵系统（10LAC10）"
      }
    ]
  }
]
```

### 5.2 查询节点详情

`GET /api/search-nav/nodes/{nodeId}`

返回结构建议直接按前端页面聚合：

```json
{
  "nodeId": "sys-1-1",
  "label": "给水泵系统（10LAC10）",
  "systemName": "给水泵系统",
  "discipline": "热机",
  "scope": "锅炉岛 / 给水泵与阀门回路",
  "owner": "专业负责人：张三",
  "description": "系统用于锅炉给水输送与压力维持。",
  "kksRoot": "10LAC10",
  "docs": [
    {
      "id": "doc-2",
      "name": "设备数据表.xlsx"
    }
  ],
  "models": [
    {
      "id": "m-1",
      "name": "锅炉系统模型"
    }
  ],
  "devices": [
    {
      "kks": "10LAC10AP001",
      "name": "主给水泵A"
    }
  ]
}
```

说明：

- `docs`、`models`、`devices` 当前都只要求最小展示字段
- 如果后续要支持右侧详情直接跳转，可额外补充 `url`、`updatedAt`、`type`

## 6. 关联关系模块接口

页面：`/search-nav/relationships`

### 6.1 查询关系类型

`GET /api/search-nav/relation-types`

返回：

```json
[
  { "label": "节点-文档", "value": "node_doc" },
  { "label": "节点-模型", "value": "node_model" },
  { "label": "KKS-文档", "value": "kks_doc" },
  { "label": "KKS-模型", "value": "kks_model" }
]
```

### 6.2 查询关系列表

`GET /api/search-nav/relations`

请求参数：

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `keyword` | string | 否 | 关键字，按来源名称和目标名称模糊匹配 |
| `type` | string | 否 | 关系类型 |

返回结构：

```json
[
  {
    "id": "r1",
    "type": "node_doc",
    "source": {
      "kind": "node",
      "id": "sys-1-1"
    },
    "target": {
      "kind": "doc",
      "id": "doc-2"
    },
    "updatedAt": "2026-01-01 10:12",
    "sourceText": "给水泵系统（10LAC10）",
    "targetText": "设备数据表.xlsx"
  }
]
```

说明：

- `sourceText`、`targetText` 建议后端直接返回，前端可直接展示
- 若后端不返回，则前端需要额外查名称映射

### 6.3 新增关系

`POST /api/search-nav/relations`

请求体：

```json
{
  "type": "node_doc",
  "source": {
    "kind": "node",
    "id": "sys-1-1"
  },
  "target": {
    "kind": "doc",
    "id": "doc-2"
  }
}
```

返回：

```json
{
  "id": "r1001"
}
```

### 6.4 编辑关系

`PUT /api/search-nav/relations/{id}`

请求体：

```json
{
  "type": "node_model",
  "source": {
    "kind": "node",
    "id": "sys-1-1"
  },
  "target": {
    "kind": "model",
    "id": "m-1"
  }
}
```

### 6.5 删除关系

`DELETE /api/search-nav/relations/{id}`

返回：

```json
{
  "success": true
}
```

## 7. 前端当前依赖的枚举和值域

### 7.1 搜索范围

```json
["documents", "models", "devices", "pid"]
```

说明：

- `pid` 仅作前端兼容占位，当前可按文档搜索处理

### 7.2 关系类型

```json
["node_doc", "node_model", "kks_doc", "kks_model"]
```

### 7.3 资源种类

```json
["node", "kks", "doc", "model"]
```

## 8. 落地建议

- 优先保证搜索接口、系统树接口、节点详情接口可用，先支撑查询场景
- 关联关系建议单独建表，避免仅依赖 `documents.nodeIds/kksRefs` 与 `models.nodeIds/kksRefs` 的冗余字段
- 若已有文档和模型主数据表，`relations` 建议作为统一关系表
- 如果后续要支持更多“搜索导航”子模块，可继续沿用 `source.kind + target.kind + type` 的关系模型扩展
