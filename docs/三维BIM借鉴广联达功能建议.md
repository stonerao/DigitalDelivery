## 一、当前三维 BIM 已具备能力

当前三维模块已经不是单纯模型预览页，已经具备三维巡视平台雏形：

| 能力 | 当前实现证据 | 说明 |
| --- | --- | --- |
| 模型浏览 | `ThreeModelViewer`、`Viewer3DWorkspace.vue` | 支持三维模型加载、旋转、平移、缩放、视角控制。 |
| 模型结构树 | `StructurePanel.vue` | 支持构件树搜索、按模型类型筛选、定位、透明、隐藏、隔离。 |
| 导航与定位 | `NavigationPanel.vue`、`NavigationMapPanel.vue` | 已有系统/KKS 定位、导航图热点、书签定位。 |
| 设备列表 | `DevicePanel.vue` | 支持设备搜索、定位、隔离。 |
| 构件属性 | `ObjectInfoPanel.vue` | 支持几何属性、自定义属性、KKS 业务属性、设备台账、结构参数、运行参数、绑定文件、测点关联。 |
| 测量 | `MeasurementPanel.vue` | 支持点距、折线、夹角、面积测量，测量结果可管理和导出。 |
| 剖切 | `ClippingPanel.vue` | 支持剖切对象、剖切方向、剖切动画、剖切状态保存。 |
| 点位与摄像头 | `SceneAnchorPanel.vue`、`CameraVideoDialog.vue` | 支持场景点位、摄像头点位、视频弹窗。 |
| 模型资产 | `AssetGroupPanel.vue`、`ModelPickerDialog.vue` | 支持场景多模型、模型选择、资源分组、质量档、LOD。 |
| 场景方案 | `SceneSchemePanel.vue` | 支持保存和恢复显隐、定位、测点显示等场景状态。 |
| 运行时联动 | `RuntimeLinkagePanel.vue` | 已有实时状态、脚本引擎、后台桥接日志入口。 |

结论：当前项目已具备 BIMFACE 类“Web BIM 浏览器”的基础能力，但与广联达完整 BIM 产品矩阵相比，还缺少协同审阅、版本追溯、4D/5D 管理、BIM+GIS 和资产生产流水线能力。

## 二、可借鉴功能总览

| 广联达能力来源 | 可借鉴功能 | 当前状态 | 建议优先级 |
| --- | --- | --- | --- |
| BIMFACE | Web BIM 浏览工具集：测量、剖切、构件树、属性、视点、截图、模型轻量化 | 大部分已具备 | P0 持续完善 |
| BIMMAKE | 模型资产生产：导模、轻量化、缩略图、场景构件库、LOD 产物 | 部分具备模型管理和 LOD 消费 | P0/P1 |
| BIM 设计协同平台 | 多专业模型协同、版本追溯、校审、交付归档 | 仅有模型/文档绑定底座 | P1 |
| 协筑 | 模型批注、问题任务、文档版本、在线预览、流程闭环 | 文档绑定有基础，问题闭环缺失 | P1 |
| BIM5D | 进度模拟、成本/物资、质量安全问题与模型关联 | 暂未形成 4D/5D | P2 |
| BIM+智慧工地 | IoT、摄像头、实时数据、告警、项目驾驶舱 | 摄像头/测点/运行时有基础 | P1/P2 |
| 数字孪生平台 | BIM+GIS、场景图层、多源空间数据、空间分析、大场景渲染 | 图层与导航有基础，GIS 缺失 | P2 |

## 三、建议优先借鉴的功能

### 1. 模型资产中心增强

对应广联达能力：BIMMAKE、BIMFACE。

当前已有：

- 模型管理列表、模型缩略图。
- 三维全屏资产选择弹窗。
- 多模型加载、质量档、LOD、资源分组。

建议增强：

- 增加模型状态：上传中、转换中、缩略图生成中、可预览、转换失败。
- 增加模型版本：版本号、上传人、上传时间、变更说明、是否当前版本。
- 增加模型专业分类：土建、机务、电气、热控、管道、厂区、GIS。
- 增加模型资产卡片字段：缩略图、文件大小、模型格式、转换状态、项目名称、最近使用时间。
- 增加模型发布包：原始文件、轻量化文件、缩略图、构件树、属性索引、LOD 资源、scene-manifest。

落地建议：

- P0：模型选择卡片化和缩略图保持一致体验。
- P0：补齐模型鉴权下载、缩略图鉴权、three.js 加载鉴权。
- P1：后端增加模型转换任务和转换状态。
- P1：模型版本管理与版本切换。

### 2. 构件级属性与业务数据卡片

对应广联达能力：BIMFACE 构件属性、BIM 设计协同平台构件级交付。

当前已有：

- 构件几何属性。
- 自定义属性。
- KKS 业务属性。
- 设备台账。
- 结构参数、运行参数。
- 绑定文件和测点。

建议增强：

- 增加属性分组：基础信息、设计属性、施工属性、运维属性、KKS、设备台账、实时数据。
- 增加属性来源：模型内属性、KKS 台账、设备台账、实时测点、人工绑定。
- 增加属性搜索和复制。
- 增加“跳转关联对象”：文档、图纸、KKS、设备详情、测点曲线、摄像头。
- 增加“缺失数据提示”：未绑定 KKS、未绑定台账、未绑定文档、无实时测点。

落地建议：

- P0：继续完善 `ObjectInfoPanel.vue`，把属性展示做成清晰分组。
- P1：为构件属性建立统一数据结构，避免前端在多个来源中临时拼字段。
- P1：支持从构件发起“绑定 KKS / 绑定文档 / 绑定测点 / 绑定摄像头”。

### 3. 模型批注与问题闭环

对应广联达能力：协筑 BIM 协作、BIM 设计协同校审。

当前已有：

- 视点书签。
- 场景点位。
- 文档绑定。
- 构件选择和定位。

缺口：

- 没有完整的问题批注模型。
- 没有问题责任人、状态、截止时间、整改附件。
- 没有按视点恢复问题现场。

建议功能：

- 在三维场景中右键构件或点位，创建问题。
- 保存问题视点：相机位置、目标点、构件 UUID、截图。
- 问题字段：标题、描述、类型、等级、责任人、截止时间、状态、整改说明、附件。
- 问题状态：待处理、处理中、待复核、已关闭、驳回。
- 支持按问题定位模型构件并恢复视角。
- 支持问题导出和统计。

落地建议：

- P1：新增 `IssueAnnotationPanel` 或在点位体系上扩展 `businessBinding.bindingType = issue`。
- P1：复用现有场景点位、书签、截图、文档绑定能力。
- P2：接入消息提醒和流程审批。

### 4. 模型版本对比与变更追溯

对应广联达能力：设计协同平台、协筑版本管理。

当前已有：

- 模型列表和项目场景保存。
- 文档绑定和场景方案保存。

缺口：

- 模型版本没有形成可追溯链路。
- 缺少版本差异对比。
- 缺少模型变更记录和发布记录。

建议功能：

- 模型版本列表：当前版本、历史版本、发布时间、发布人、变更说明。
- 模型版本切换：在场景中切换同一模型不同版本。
- 简化版版本对比：新增构件、删除构件、变更构件数量统计。
- 视图层版本对比：同屏加载两个版本，用颜色区分新增、删除、变更。
- 发布记录：记录模型、文档、KKS、点位、摄像头等交付物版本。

落地建议：

- P1：先做模型版本元数据和历史版本切换。
- P2：再做几何差异和构件属性差异对比。

### 5. 图纸/文档与模型联动审阅

对应广联达能力：协筑文档协同、BIM 设计协同数字化交付。

当前已有：

- 构件绑定文件。
- 文档预览组件。
- 模型对象与文档关系服务。

建议增强：

- 在对象属性面板中直接预览绑定文档。
- 支持图纸页码/图纸区域与构件绑定。
- 支持从文档批注反向定位模型构件。
- 支持文档版本对比和版本历史。
- 支持文档、模型、KKS 三方关联关系一键查看。

落地建议：

- P1：对象属性面板增加“打开预览”操作。
- P1：关系模块增加 `model_object_doc` 的预览入口和批注入口。
- P2：支持图纸区域标注与构件定位。

### 6. 4D 进度模拟

对应广联达能力：BIM5D 施工模拟、进度控制。

当前已有：

- 场景方案。
- 构件显隐。
- 资源分组。
- 运行时脚本联动。

缺口：

- 没有 WBS/计划任务。
- 没有构件与进度任务绑定。
- 没有时间轴播放。

建议功能：

- 建立 WBS 任务树。
- 将任务绑定模型构件、资源分组或系统节点。
- 时间轴播放施工/安装/调试过程。
- 颜色表达状态：未开始、进行中、已完成、延期、风险。
- 支持按日期查看模型状态。

落地建议：

- P2：先实现“任务 + 构件 UUID/分组绑定 + 时间轴显隐”。
- P2：后续接入项目管理计划数据。

### 7. 5D 成本与物资统计

对应广联达能力：BIM5D 成本、物资、三算对比。

当前已有：

- KKS 台账。
- 设备台账。
- 模型属性和构件树。

缺口：

- 没有工程量提取。
- 没有成本科目。
- 没有物资台账与构件绑定。

建议功能：

- 设备/材料清单与构件绑定。
- 按系统、区域、模型、KKS 统计数量。
- 物资到货、安装、验收状态在模型中着色。
- 支持按构件查看成本、合同、供应商、到货批次。

落地建议：

- P2：先做台账统计视图和模型着色。
- P2：不要一开始做完整成本管理，先打通“构件 - KKS - 设备/材料”关系。

### 8. 智慧工地/运维实时数据联动

对应广联达能力：BIM+智慧工地、IoT 接入、数据决策。

当前已有：

- 测点关联。
- 摄像头点位。
- 运行时联动面板。
- 运行参数展示。

建议增强：

- 设备实时状态在模型中着色：正常、预警、告警、离线。
- 点击测点查看历史曲线。
- 摄像头支持实时预览、回放、关联设备。
- 告警列表与三维模型联动定位。
- 增加项目级驾驶舱：设备总数、告警数、在线率、未处理问题数。

落地建议：

- P1：模型对象状态着色和告警定位。
- P1：测点历史曲线入口。
- P2：统一 IoT 数据源接入和驾驶舱。

### 9. BIM+GIS 与全厂级数字孪生

对应广联达能力：数字孪生平台、BIM+GIS。

当前已有：

- 平面渲染。
- 导航图。
- 场景图层和资源分组。
- 点位/摄像头坐标。

缺口：

- 没有 GIS 底图。
- 没有坐标系转换。
- 没有厂区级空间数据图层。
- 缺少大场景按需加载。

建议功能：

- 厂区总览图或 GIS 底图。
- BIM 模型与厂区坐标系绑定。
- 建筑、道路、管线、设备、摄像头、传感器按图层管理。
- 支持区域框选、空间查询、距离/面积测量。
- 支持按区域加载模型和卸载模型。

落地建议：

- P2：先做厂区导航图和区域图层。
- P2：再接入 GIS 地图与坐标转换。
- P2：大模型场景需要瓦片化、LOD、实例化和按需加载。

### 10. 场景配置与交付验收矩阵

对应广联达能力：数字化交付、设计协同交付归档。

当前已有：

- 场景保存。
- 场景方案。
- 模型、点位、摄像头、测量、剖切配置。

建议增强：

- 增加场景交付清单：模型、文档、KKS、设备、点位、摄像头、图层、视点。
- 增加完整性检查：是否绑定项目、是否有模型、是否有导航图、是否有关键设备台账。
- 增加全厂资产覆盖矩阵：资产类型、是否建模、是否入场景、是否可定位、是否绑定业务数据。
- 增加交付包导入/导出：scene-manifest、模型清单、点位、摄像头、关系数据。

落地建议：

- P0：输出资产覆盖清单文档或页面。
- P1：做交付检查页面。
- P1：保存项目时提示缺失项。

### P2：后续扩展

| 功能 | 理由 | 前置条件 |
| --- | --- | --- |
| 4D 进度模拟 | 借鉴 BIM5D 进度管理 | 需要 WBS 和构件绑定。 |

| AI 助理 | 借鉴智慧工地 AI 能力 | 需要文档、模型、KKS 数据统一索引。 |

## 五、推荐实施顺序

### 第一步：把当前三维 BIM 做成稳定交付底座

- 模型加载鉴权稳定。
- 模型资产卡片和缩略图体验统一。
- 模型、文档、KKS、设备、点位、摄像头关系可查。
- 对象属性面板完整展示台账、运行参数、绑定资料。
- 输出全厂资产覆盖矩阵。

### 第二步：补协同审阅能力

- 模型批注。
- 问题任务。
- 视点恢复。
- 整改附件。
- 问题统计和导出。

### 第三步：补交付追溯能力

- 模型版本。
- 文档版本。
- 关系版本。
- 场景发布记录。
- 交付完整性检查。

### 第四步：再扩展 BIM5D 和数字孪生

- 进度时间轴。
- 构件状态着色。
- 成本/物资统计。
- BIM+GIS。
- IoT 数据驾驶舱。

## 六、结论

当前三维 BIM 最值得优先借鉴广联达的不是“重新做一个庞大的 BIM5D”，而是先补齐三类能力：

1. 模型资产生产和交付追溯能力，来源于 BIMMAKE、BIMFACE、BIM 设计协同。
2. 构件级协同审阅和问题闭环能力，来源于协筑和设计校审。
3. 实时状态联动和场景运营能力，来源于 BIM+智慧工地和数字孪生平台。

短期建议围绕 P0/P1 做：模型资产、属性卡片、批注问题、版本追溯、文档联动、告警定位。4D 进度、5D 成本、BIM+GIS、AI 助理适合放到后续阶段，避免当前三维平台在基础数据关系尚未稳定时过早扩张。
