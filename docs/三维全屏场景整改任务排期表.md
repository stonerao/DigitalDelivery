# 三维全屏场景整改任务排期表

更新时间：2026-04-24

用途：用于项目排期、周会跟踪、里程碑管理。

说明：

- 本排期为建议版，基于当前问题清单和整改实施计划推导。
- 默认前提为：前端、后台、模型/资产、项目管理四类角色均可参与。
- 若当前项目实际资源不足，需要优先保障 `P0` 项。

## 一、排期目标

本排期分三阶段推进：

1. 第一阶段：先完成验收收口项
2. 第二阶段：补齐体验和业务展示项
3. 第三阶段：推进中长期能力建设项

建议以 2026-04-27 作为排期起点。

## 六、风险与缓冲建议

| 风险项 | 风险描述 | 建议处理方式 |
| --- | --- | --- |
| 范围不清 | 局部示范与全厂交付口径混淆 | 第 1 周内先明确范围，不明确则不进入正式验收口径 |
| 数据不统一 | 台账字段和参数字段来源不一致 | 先输出字段白名单，再做前端接入 |
| 资产清单不完整 | 模型侧无法快速输出全厂资产台账 | 先按区域分批整理，优先补齐需求中明确列出的资产类别 |
| 导航图素材缺失 | 缺少统一厂区总图或区域图 | 优先使用业务侧确认的总图，避免前端临时拼图 |
| 流水线周期长 | P2 涉及模型、工程化、发布协同 | 与 P0/P1 分阶段推进，避免阻塞短期验收目标 |

## 七、状态更新建议

建议后续统一用以下状态字段维护排期：

- `未开始`
- `进行中`
- `已完成`
- `阻塞`
- `延期`

建议每周更新以下内容：

- 本周状态
- 本周完成结果
- 当前阻塞项
- 是否影响里程碑

## 八、建议使用方式

1. 项目经理在周会上直接按本排期表过任务状态。
2. 前端、后台、模型侧分别补充各自任务说明。
3. 若本期目标只保 P0，则直接冻结 P1/P2 时间，避免范围膨胀。
