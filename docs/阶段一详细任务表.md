# BIM三维平台阶段一详细任务表

日期：2026-03-25

## 1. 文档目标

本文档用于细化 BIM 三维平台“阶段一：运行时架构拆分”的实施任务、交付物、依赖关系、验收标准与建议顺序。

对应文档：

- [技术分册.md](e:/项目/数字交付/DigitalDelivery/docs/技术分册.md#L1)
- [运行时架构拆分设计.md](e:/项目/数字交付/DigitalDelivery/docs/运行时架构拆分设计.md#L1)

## 2. 阶段目标

阶段一不新增大功能，目标是为后续点位、摄像头、测量扩展打基础。

阶段目标包括：

- 降低 [fullscreen.vue](e:/项目/数字交付/DigitalDelivery/src/views/visualization/viewer3d/fullscreen.vue#L1) 的复杂度
- 建立运行时 store
- 建立 Viewer 适配层
- 将主要面板与工具栏组件化
- 将复杂业务逻辑从页面中迁移到 service 层
- 保持现有功能行为基本不变

## 3. 拆分范围

本阶段包含：

- 页面结构拆分
- 状态拆分
- Viewer 调用封装
- 服务层提取
- 基础回归验证

本阶段不包含：

- 点位编辑功能开发
- 摄像头视频接入
- 新测量工具实现
- 实时数据接入
- 脚本引擎开发

## 4. 任务清单总览

建议拆成 8 个主任务。

| 编号 | 任务 | 目标 |
|---|---|---|
| T1 | 梳理现有状态与命令 | 形成迁移底稿 |
| T2 | 建立运行时 store | 收拢页面状态 |
| T3 | 建立 ViewerAdapter | 封装 viewer 调用 |
| T4 | 抽出基础 composable | 统一初始化与销毁 |
| T5 | 拆分顶部与底部工具组件 | 降低主页面模板复杂度 |
| T6 | 拆分对象属性与剖切面板 | 迁移局部状态与交互 |
| T7 | 拆分导航、结构、设备、场景方案面板 | 完成右侧面板模块化 |
| T8 | 抽取 service 层并做回归验证 | 固化运行时架构 |

## 5. 详细任务说明

## T1 梳理现有状态与命令

目标：

- 识别哪些状态属于页面展示、哪些属于运行时、哪些属于工程数据
- 识别所有直接调用 `viewerRef` 的入口点

主要工作：

- 盘点 `ref/computed/watch/function`
- 标记与三维内核交互的函数
- 标记可迁移到 store 的状态
- 标记可迁移到 service 的流程

输出：

- 状态清单
- viewer 命令清单
- 页面函数归类清单

验收标准：

- 能明确每个状态后续归属位置
- 能明确每个 `viewerRef.value?.xxx` 的迁移目标

## T2 建立运行时 store

目标：

- 将运行时状态从页面中迁移到 store

主要工作：

- 新建 `viewer3dRuntime.js`
- 新建 `viewer3dProject.js`
- 将工具、面板、显示模式、选择态迁移到 runtime store
- 将场景方案、设备索引、后续点位数据入口迁移到 project store

输出：

- `src/store/modules/viewer3dRuntime.js`
- `src/store/modules/viewer3dProject.js`

验收标准：

- 页面中大量 `ref` 被收敛
- 工具状态和面板状态能通过 store 正常响应

依赖：

- 无

## T3 建立 ViewerAdapter

目标：

- 页面和业务逻辑不再直接访问 `viewerRef`

主要工作：

- 新建 `viewerAdapter.js`
- 定义 `bindViewer/unbindViewer`
- 封装模型、选择、显隐、剖切、测量、截图、书签、漫游等命令
- 对底层事件进行统一转发

输出：

- `src/3d/runtime/adapter/viewerAdapter.js`

验收标准：

- 页面中不再新增直接 `viewerRef.value.xxx`
- 主要逻辑通过适配器调用 viewer

依赖：

- T1

## T4 抽出基础 composable

目标：

- 将 viewer 运行时初始化、销毁、事件绑定逻辑从页面中抽离

主要工作：

- 新建 `useViewerRuntime.js`
- 负责：
  - 路由模型解析
  - viewer 绑定
  - store 初始化
  - 生命周期清理
  - 运行时事件订阅

输出：

- `src/views/visualization/viewer3d/composables/useViewerRuntime.js`

验收标准：

- 页面中的 `onMounted/onBeforeUnmount/watch` 明显减少
- 初始化流程集中可读

依赖：

- T2
- T3

## T5 拆分顶部与底部工具组件

目标：

- 将模板里高频但边界清晰的 UI 先抽出去

主要工作：

- 抽出顶部栏组件
- 抽出底部工具栏组件
- 使用 store 和 service 驱动按钮行为

输出：

- `ViewerTopbar.vue`
- `ViewerBottomToolbar.vue`

验收标准：

- 主页面模板显著缩短
- 工具切换、画质切换、透明、投影、剖切、截图、书签功能不回退

依赖：

- T2
- T3

## T6 拆分对象属性与剖切面板

目标：

- 将左下和右上相对独立的面板先组件化

主要工作：

- 抽出对象属性面板
- 抽出剖切控制面板
- 把相关状态与方法迁移到 store/service

输出：

- `ObjectInfoPanel.vue`
- `ClippingPanel.vue`

验收标准：

- 对象拾取后面板能正常展示
- 剖切开关和三轴区间控制行为不变

依赖：

- T2
- T3

## T7 拆分导航、结构、设备、场景方案面板

目标：

- 完成右侧复合侧栏的模块化

主要工作：

- 将导航、结构、设备、场景方案视图拆成独立组件
- 将显隐树、定位、隔离、筛选、场景方案行为搬到 service

输出：

- `NavigationPanel.vue`
- `StructurePanel.vue`
- `DevicePanel.vue`
- `SceneSchemePanel.vue`

验收标准：

- 右侧面板切换行为正常
- 导航定位、结构树定位、设备列表定位、场景方案应用功能正常

依赖：

- T2
- T3
- T4

## T8 抽取 service 层并做回归验证

目标：

- 固化运行时分层，完成阶段收口

主要工作：

- 提取：
  - `selectionService`
  - `visibilityService`
  - `schemeService`
  - `clippingService`
- 清理页面中残留的大型函数
- 进行回归检查

输出：

- service 文件集合
- 阶段一回归记录

验收标准：

- `fullscreen.vue` 以编排为主
- 复杂逻辑主要进入 service
- 已有功能可回归通过

依赖：

- T2
- T3
- T5
- T6
- T7

## 6. 推荐实施顺序

建议顺序如下：

1. T1 梳理现状
2. T2 建立 store
3. T3 建立 ViewerAdapter
4. T4 抽出 composable
5. T5 拆顶部与底部工具栏
6. T6 拆对象属性与剖切面板
7. T7 拆右侧侧栏面板
8. T8 抽 service 并回归

原因：

- 先收拢状态和命令，再拆 UI，返工最少
- service 层应该在组件拆分后抽取，边界更清晰

## 8. 建议测试重点

阶段一虽然不做新功能，但必须盯住回归。

重点验证：

- 模型加载与切换
- 工具切换
- 漫游切换
- 对象拾取与属性展示
- 系统定位、区域定位、KKS 定位
- 显隐树勾选
- 场景方案保存与应用
- 剖切与测量基础能力

## 9. 风险与注意事项

主要风险：

- UI 拆分后事件链过长
- store 拆分不当导致状态分散
- viewer 适配层不彻底，页面继续绕过适配器
- 迁移过程中 watch 行为丢失

注意事项：

- 拆分时保持“行为不变”
- 每完成一个组件拆分就做一次局部验证
- 不建议在阶段一同时引入点位系统代码
- `fullscreen.vue` 中用户已有交互逻辑必须逐项迁移，不要一次性删除

## 10. 完成标志

满足以下条件可标记阶段一完成：

- `fullscreen.vue` 中模板与逻辑显著缩减
- store、adapter、service、panel 四类模块已建立
- 关键功能通过回归
- 已能承接阶段二点位系统接入而不再回到大页面堆逻辑
