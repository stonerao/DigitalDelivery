# ConfigurationEditor（工业组态编辑器）验收测试清单 & 修改计划

日期：2026-01-19

## 1. 目标与范围
- 目标：以“可交付商业软件”的口径验收编辑器：体验顺畅、功能闭环、关键数据不丢、稳定可用。
- 范围：packages/ConfigurationEditor/src
- 重点：用户体验、功能完整性、交互一致性、性能与稳定性。

## 2. 可执行验收用例（按优先级）

### P0（上线门槛：必须全部通过）

#### A. 首次使用与基础可用性
- A1 首屏加载稳定
  - 步骤：打开编辑器→等待 10 秒→缩放 90%/110%→拖动窗口尺寸
  - 预期：布局稳定无白屏/重叠；无持续报错。
- A2 核心路径闭环
  - 步骤：拖拽组件到画布→点击选中→右侧修改名称/位置→取消选中→再次选中
  - 预期：画布即时更新；回显一致；未选中有提示。

#### B. 组件放置
- B1 拖拽落点与生成可见
  - 步骤：拖拽 5 次到不同区域（含边缘）
  - 预期：都能生成且可见；落点合理。

#### C. 属性编辑的容错
- C1 非法输入不破坏场景
  - 步骤：位置/缩放输入空值、字母、极大值、负数→失焦
  - 预期：有提示或纠正；不出现 NaN/渲染异常。

#### D. 节点列表与层级排序
- D1 列表-画布联动
  - 步骤：列表点选→画布选中；画布点选→列表选中
  - 预期：双向一致。
- D2 上移/下移/置顶/置底稳定
  - 步骤：创建 10 节点→各类排序操作各 5 次
  - 预期：无报错；层级立即生效；不丢选中。

#### E. 数据安全（破坏性操作 + dirty）
- E1 删除确认一致
  - 步骤：画布右键删除、节点列表右键删除、Delete 删除
  - 预期：确认策略一致；取消不产生变更；删除可撤销。
- E2 未保存更改保护
  - 步骤：制造 dirty→新建/导入→分别取消/放弃
  - 预期：取消不丢数据；放弃后正确清空并复位。

#### F. 工程导入/导出/加载一致性（最关键）
- F1 导出→刷新→导入一致
  - 步骤：构建包含：节点/分组/层级/状态/事件/数据源/绑定/背景/视角 的场景→导出→刷新→导入
  - 预期：全部一致；尤其数据源与绑定不丢。
- F2 错误文件处理
  - 步骤：导入损坏 JSON/旧版本 JSON
  - 预期：错误提示明确；不会半加载污染当前工程。

### P1（体验与交付效率：建议进入最近迭代）

#### G. 事件系统
- G1 节点事件：创建-保存-触发
  - 步骤：为节点创建 click 事件并配置动作→预览触发
  - 预期：动作执行一次且可见；能定位触发了哪个事件。

#### H. 数据源联通性
- H1 API 数据源状态与错误提示
- H2 WebSocket 心跳/重连策略可控

#### I. 性能与稳定性
- I1 500/2000 节点基线：缩放/平移/搜索/层级调整
- I2 长时间编辑 30 分钟无累积性异常

### P2（增强项：差异化/商业化）
- 模板/行业资产体系
- 权限/协作/发布链路

## 3. 修改计划（按迭代拆分）

### 迭代 1（P0：数据不丢 + 安全感）
